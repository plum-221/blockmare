# 《夜行 · BLOCKMARE — 长夜》设计文档

> 技术栈：Three.js r160（本地引用）+ 纯 WebAudio 程序化音频 + 单文件 index.html。
> 部署：GitHub Pages → https://plum-221.github.io/blockmare/
> 目标：短篇叙事生存恐怖（1–2 小时），保留体素 + 第一人称 + 资源管理 + 实体追逐核心循环。

## 一、世界观框架《长夜》
第七日，太阳没有升起，村庄陷入永夜。黑暗里出现「它」——一个一直凝视村民的人影，
随火把熄灭逐步逼近。村民为换回太阳而献祭、背叛、封印失败，最终被黑暗吞噬，村庄成废墟。
玩家是少数清醒者，要集齐被封存的村民残魂（4 块灵魂碎片），回到出生点的「黎明祭坛」点亮它，
带来黎明 = 逃脱。剧情通过散落的石碑/日记一段段揭开。

## 二、4 块碎片对应的村民悲剧（环境叙事）
1. **磨坊主·老格雷** —— 主张献祭孩子换太阳，亲手送走女儿。
2. **铁匠之妻·莉娜** —— 丈夫去重封失败，它用丈夫的嗓音在门外喊她，她钉死了门。
3. **牧师·班** —— 封印需七个真名，他手抖把第七个写成了「你」的名字（暗示玩家与封印的关系）。
4. **孩子·萨米** —— 唯一不怕它的人，被它以「一起永远玩」诱骗，最先沉睡。

每块碎片旁立一块石碑，走近 E 阅读即解锁该村民往事；出生点另有一块「序章碑」。

## 三、多结局系统（Phase 3 实装，条件已预埋 storiesRead）
- **黎明结局（Good）**：集齐碎片 + 读完全部 4 段往事 + 理智 > 阈值 → 真正驱散黑暗。
- **逃离结局（Neutral）**：集齐碎片点亮祭坛，但未读完往事 → 逃出但真相未明。
- **沉沦结局（Bad）**：理智归零 → 成为它的一部分。
- **献祭结局（Secret）**：在祭坛做出某种选择（待设计）→ 用自己换黎明。

## 五、技术决策记录
- **音频走程序化合成而非下载素材**：单文件零依赖、即时加载、无版权风险、体积小（GitHub Pages 友好）。
  位置感用 WebAudio PannerNode(HRTF) 实现，真·人声低语用浏览器内置 SpeechSynthesis。
  后续若要更丰富的 dark ambient 底噪，再考虑内联 1–2 个 CC0 ogg loop（base64 或 assets/）。
- **性能优先**：体素用 InstancedMesh，按方块类型分组。**网格按 8×8 列分区块**，编辑只重建受影响区块
  （自身 + 跨边界的 x/z 邻区块）；实测较旧版整表重建快 ~64×（19.3ms→0.3ms/次），消除挖放掉帧。
  正确性以"分区暴露面集合 === 全扫描参考集合"不变量校验（含区块缝）。
- **素材来源（如将来需要）**：音效 freesound.org / pixabay；纹理 Kenney.nl / Poly Haven(CC0)。
  目录建议：`assets/audio/`、`assets/textures/`，用 fetch/AudioBuffer 加载。当前全程序化，暂不需要。
